# RetroPick Hackathon Demo — Start to Finish

Run the deterministic, no-LLM CRE demo from scratch.

---

## 1. Prerequisites

- **CRE CLI** installed
- **Bun** (1.2.21+)
- **CRE account** — `cre login` if needed
- **Sepolia private key** — for signing (simulation works without broadcast)

---

## 2. Clone & Install

```bash
# From repo root (sc-cre-workflow-chainlink/)
cd apps/workflow
bun install
```

`bun install` runs `cre-setup` and creates `tmp.wasm` in `src/`.

---

## 3. Configure

### 3.1 Create `.env`

In `apps/workflow/`:

```bash
# apps/workflow/.env
CRE_ETH_PRIVATE_KEY=0xYOUR_64_CHAR_PRIVATE_KEY
RPC_URL=https://ethereum-sepolia-rpc.publicnode.com
```

### 3.2 Update `config.demo.json`

In `apps/workflow/config.demo.json`, set:

- `evms[0].marketAddress` — PoolMarketLegacy (for settlement log trigger)
- `creReceiverAddress` — CREReceiver (for settlement reports)
- `creatorAddress` — your wallet
- `curatedPath.crePublishReceiverAddress` — for publish-from-draft (optional)

Use placeholder `0x0000...` if you only want to test proposal preview.

---

## 4. Run Demo (Non-Interactive)

**Important:** Run from the **monorepo root** (`sc-cre-workflow-chainlink/`). The CRE CLI looks for `workflow.yaml` relative to the project root; running from `apps/workflow` with `.` causes "workflow.yaml: no such file or directory".

```bash
cd /path/to/sc-cre-workflow-chainlink   # monorepo root
cre workflow simulate ./apps/workflow -T demo-settings \
  -e apps/workflow/.env \
  --non-interactive \
  --trigger-index 0 \
  --http-payload @demo/fixtures/proposal-safe.json
```

- `--trigger-index 0` — HTTP trigger (1 = EVM log)
- `--non-interactive` — no prompts, uses `--http-payload` directly
- `-e apps/workflow/.env` — env file (needed when running from monorepo root)
- `@demo/fixtures/...` — payload path is **relative to the workflow folder** (`apps/workflow`), not the project root

### Expected Output

```
Workflow compiled
🚀 Workflow simulation ready...
...
Workflow Simulation Result:
 {"ok":true,"policy":{"status":"ALLOW",...},"draftId":"0x3c37...","status":"PENDING_CLAIM",...}
```

## 5. Run Demo (Interactive)

From monorepo root:

```bash
cre workflow simulate ./apps/workflow -T demo-settings -e apps/workflow/.env
```

When prompted:

1. **Select trigger:** `1` (HTTP)
2. **Enter input:** `@demo/fixtures/proposal-safe.json` or paste the JSON:

```json
{"title":"Will ETH exceed $6000 by December 31, 2026?","body":"Threshold market based on public market data","sourceType":"http_proposal"}
```

## 7. Why Not From `apps/workflow`?

Running `cre workflow simulate .` from `apps/workflow` **fails** with `workflow.yaml: no such file or directory`. The CRE CLI discovers the project root (where `project.yaml` lives) and resolves the workflow path relative to it; `.` then points to the project root, which has no `workflow.yaml`. Always run from the monorepo root with `./apps/workflow`.

---

## 8. Troubleshooting

| Error | Fix |
|-------|-----|
| `workflow.yaml: no such file or directory` | Run from **monorepo root** with `./apps/workflow`, not from `apps/workflow` with `.` |
| `no RPC URLs found` | Add `demo-settings` to `project.yaml` (monorepo root) with `rpcs` for Sepolia |
| `open demo/tmp.wasm: no such file` | Demo uses `src/demo-main.ts`; run `bun install` in `apps/workflow` |
| `failed to get HTTP trigger payload: empty input` | Use `--non-interactive --trigger-index 0 --http-payload @path/to/file.json` |
| `failed to read file .../proposal-safe.json` | Use `@demo/fixtures/proposal-safe.json` (path relative to workflow folder) or absolute path |
| `Question is required` / `Received question: undefined` | Fixtures must use `question` (not `title`). Update `proposal-review.json` and `proposal-reject.json` to have `"question": "..."` instead of `"title": "..."`. Or run `bun install` in `apps/workflow` to rebuild with the handler that accepts both. |
| `Missing creReceiverAddress` | Set in `config.demo.json` (or use placeholder for proposal-only tests) |
